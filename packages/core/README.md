# `lets-config`

Framework-agnostic core for `@runtime-config` — inspired by Nuxt 3/4's `useRuntimeConfig`.

## API

### `defineRuntimeConfig(config)`
Identity helper with full TypeScript inference.

### `applyEnvOverrides(config, env?, prefix?)`
Deep-merges matching env vars onto a config object.

**Convention** (prefix defaults to `RUNTIME_`):
| Env var | Config path |
|---|---|
| `RUNTIME_PUBLIC_API_BASE` | `config.public.apiBase` |
| `RUNTIME_DB_URL` | `config.dbUrl` |
| `RUNTIME_PUBLIC_FEATURE_FLAGS__DARK_MODE` | `config.public.featureFlags.darkMode` |

Use `__` as the nesting separator, `_` as the camelCase word separator.

### `serializePublicConfig(config)` / `buildConfigScript(config)`
Serialise the `public` portion to a JSON string / inline `<script>` tag.

## Type augmentation

```ts
// src/runtime-config.d.ts  (or auto-generated by @lets-config/vite)
declare module 'lets-config' {
  interface PrivateRuntimeConfig {
    dbUrl: string
    secretKey: string
  }
  interface PublicRuntimeConfig {
    apiBase: string
    appName: string
  }
}
```
